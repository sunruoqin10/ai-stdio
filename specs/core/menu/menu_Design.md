## 1. 设计原则

### 1.1 核心原则

- **清晰层级**: 通过树形结构清晰展示菜单层级关系
- **高效操作**: 减少操作步骤,提供快捷操作方式
- **即时反馈**: 所有操作都有明确的视觉反馈
- **一致性**: 与系统整体设计风格保持一致

### 1.2 设计目标

- 提供直观的菜单管理界面
- 支持快速创建和编辑菜单
- 清晰展示菜单层级关系
- 友好的图标选择体验
- 流畅的交互体验

## 6. 响应式设计

### 6.1 断点定义

```css
/* 超小屏 */
@media (max-width: 768px) { }

/* 小屏 */
@media (min-width: 768px) and (max-width: 992px) { }

/* 中屏 */
@media (min-width: 992px) and (max-width: 1200px) { }

/* 大屏 */
@media (min-width: 1200px) { }
```

### 6.2 响应式布局

**超小屏 (< 768px)**:
- 表单弹窗: 95% 宽度
- 表格: 隐藏非核心列
- 操作按钮: 折叠到下拉菜单

**小屏 (768px - 992px)**:
- 表单弹窗: 90% 宽度
- 表格: 隐藏次要列
- 操作按钮: 主要按钮显示

**中屏 (992px - 1200px)**:
- 表单弹窗: 85% 宽度
- 表格: 全部列显示
- 操作按钮: 全部显示

**大屏 (> 1200px)**:
- 表单弹窗: 800px 固定宽度
- 表格: 全部列显示
- 操作按钮: 全部显示

### 6.3 表格响应式

**隐藏列策略**:

| 屏幕尺寸 | 隐藏列 |
|---------|--------|
| < 768px | 创建时间, 可见, 排序 |
| 768px - 992px | 创建时间 |
| > 992px | 全部显示 |

**横向滚动**:
```
容器: overflow-x: auto
最小宽度: 1200px
滚动条: 自定义样式
```
