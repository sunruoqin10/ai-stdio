## 🎯 三层架构速览

```
┌─────────────────────────────────────────────────┐
│  第一层: 功能需求规范 (Functional Spec)         │
│  📝 核心问题: 做什么?                            │
│  👥 目标读者: 产品、业务、设计、开发              │
├─────────────────────────────────────────────────┤
│  ✅ 功能概述                                     │
│  ✅ 用户故事                                     │
│  ✅ 功能清单                                     │
│  ✅ 交互流程                                     │
│  ✅ 业务规则                                     │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│  第二层: 技术实现规范 (Technical Spec)          │
│  💻 核心问题: 怎么做?                            │
│  👥 目标读者: 前端、后端、测试工程师              │
├─────────────────────────────────────────────────┤
│  ✅ 数据结构                                     │
│  ✅ API接口                                      │
│  ✅ 验证规则                                     │
│  ✅ 算法实现                                     │
│  ✅ 自动化功能                                   │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│  第三层: UI/UX规范 (Design Spec)                │
│  🎨 核心问题: 长什么样?                          │
│  👥 目标读者: UI/UX设计师、前端工程师            │
├─────────────────────────────────────────────────┤
│  ✅ 页面布局                                     │
│  ✅ 组件选择                                     │
│  ✅ 交互规范                                     │
│  ✅ 样式规范                                     │
│  ✅ 响应式设计                                   │
└─────────────────────────────────────────────────┘
```

## 💡 使用建议

### 场景1: 创建新模块规范
```bash
# 使用新模板
cp specs/_template/spec-template-v2.md specs/[category]/[module]-spec-v2.md
```

### 场景2: AI辅助开发
```bash
# 提示词示例
"根据 specs/core/employee-spec-v2.md 三层架构规范:
- 第一层: 理解业务需求(用户故事、功能清单、业务规则)
- 第二层: 实现技术方案(数据结构、API接口、算法)
- 第三层: 设计UI界面(布局、组件、样式)
生成完整的员工管理模块代码"
```

### 场景3: 团队协作
- **产品经理**: 只看第一层,确认需求
- **后端工程师**: 重点看第一层、第二层
- **前端工程师**: 重点看第二层、第三层
- **UI设计师**: 重点看第一层、第三层
